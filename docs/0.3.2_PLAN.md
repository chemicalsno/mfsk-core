# 0.3.2 plan вҖ” uvpacket AFC for SSB use

> Status: design sketch. Implementation deferred until 0.3.1 ships.

## Motivation

The 0.3.1 modem assumes `audio_centre_hz` is known precisely.
This is fine for NFM (no frequency offset between TX/RX) and for
laboratory use, but real SSB receivers carry a Вұ50вҖ“100 Hz centre
offset from VFO-dial mismatches between TX and RX (mechanical
dials are worse). Without AFC, both ends must agree on dial
frequency to within ~10 Hz вҖ” too tight for hand-tuned operation.

With AFC, uvpacket runs at its true вҲ’3.7 dB SNR_3kHz Robust
threshold on SSB, opening up HF weak-signal data and microwave
SSB applications.

## Approach: FFT-based frequency search at the preamble

The preamble correlator is naturally a coherent integrator over
the 31 m-sequence chips. With a frequency offset О”f, the chip-
multiplied samples rotate at О”f:

```
x_i = s_i^* В· mf_out[off + iВ·NSPS]
    = signal_i В· s_i^* В· e^{jВ·2ПҖВ·О”fВ·iВ·T_sym} + noise
    = (constant signal magnitude) В· e^{jВ·2ПҖВ·О”fВ·iВ·T_sym} + noise
```

So `ОЈ_i x_i В· e^{вҲ’jВ·2ПҖВ·fВ·iВ·T_sym}` is a DFT over the preamble's
chip-rate samples. Peaking that magnitude over `f` gives О”f with
no extra MF computation.

### Resolution

- `N_chip = 31`, `T_sym = 1/1200 s` вҶ’ natural DFT bin width
  `1 / (31 В· T_sym) = 38.7 Hz`.
- Zero-pad to e.g. `N = 256` вҶ’ bin width `1200 / 256 = 4.7 Hz`.
- Coverage: `Вұfs_sym / 2 = Вұ600 Hz` (cyclic). Plenty for SSB
  Вұ200 Hz worst-case.

### Algorithm sketch (in `decode_known_layout_with_afc`)

1. Down-convert + MF once at nominal `audio_centre_hz` (existing).
2. Run integer-sample preamble correlation (existing) to lock
   onto `best_off`, but **without** the parabolic timing
   refinement yet вҖ” that depends on О”f.
3. Build the 31-element complex array
   `x_i = mf_out[best_off + iВ·NSPS] В· (preamble_bpsk[i] ? -1 : +1)`.
4. Zero-pad to `N = 256` (or similar power of 2).
5. FFT вҶ’ magnitude spectrum.
6. Argmax over `Вұ200 Hz` window (only the bins inside the
   configured search range are candidates).
7. Optional parabolic refinement within the FFT bin spacing.
8. Convert bin index вҶ’ `О”f` in Hz.
9. **Re-do** the down-convert + MF at `audio_centre_hz + О”f`,
   then continue with the existing parabolic-timing,
   amplitude / Пғ_n estimation, LMS phase fit, etc.

The "re-do" step is the cost: one extra full MF pass. With
`audio_len вүҲ 5000` and `RRC_LEN = 61`, that's ~300k complex
multiplies вҖ” single-digit-ms in release mode.

### API

```rust
pub struct AfcOpts {
    /// One-sided search range in Hz. Вұ200 Hz covers typical SSB
    /// VFO mismatch worst-case.
    pub search_hz: f32,
    /// FFT zero-pad length. Larger = finer resolution but more
    /// CPU; 256 gives 4.7 Hz steps.
    pub fft_n: usize,
}

impl Default for AfcOpts {
    fn default() -> Self {
        Self { search_hz: 200.0, fft_n: 256 }
    }
}

pub fn decode_known_layout_with_afc(
    audio: &[f32],
    sample_offset: usize,
    audio_centre_hz: f32,
    mode: Mode,
    n_blocks: u8,
    fec_opts: &FecOpts,
    afc_opts: &AfcOpts,
) -> Result<DecodedFrame, DecodeError>;
```

The default `decode_known_layout` keeps its current zero-AFC
behaviour for backward compatibility. SSB callers explicitly opt
in via `_with_afc`.

## Known unknowns

- **Doppler-shift drift over the burst** вҖ” for fast HF fading or
  microwave EME, the carrier may drift by tens of Hz across the
  ~270 ms Express / 440 ms Robust burst. The current LMS
  quadratic phase fit already absorbs slow phase drift; whether
  it copes with frequencies that the AFC corrected to "near zero"
  but still drifting is an empirical question. Likely fine
  unless the Doppler rate exceeds ~10 Hz/s, which is rare.
- **Multiple bursts at different frequencies** вҖ” the auto-detect
  `decode()` path currently scans for preamble correlation peaks
  at the nominal frequency only. SSB users with multiple
  simultaneous frames at different audio centres would need a
  2-D scan (offset Г— frequency). Out of scope for 0.3.2's
  initial AFC; covered by stating "one frame per burst" in the
  AFC API contract.

## Out of scope

- Adaptive AFC across the burst (frequency tracking). LMS phase
  fit already captures slow drift.
- Multi-carrier (multiple QPSK frames at different frequencies in
  the same audio). The auto-detect path could be extended later.
