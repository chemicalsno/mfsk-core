# 0.3.3 plan — multi-channel SSB receive + slotted ALOHA TX

> Status: design draft. Builds on 0.3.2 AFC.

## 1. Operating concept — slotted ALOHA in audio frequency

A private group shares a single RF SSB channel (e.g.,
430.090 MHz USB). Every member's transceiver is dialed to that
exact RF frequency. Inside the SSB audio passband, the modem
recognises a **slot grid** at 1200 Hz spacing — typically 800 Hz
and 2000 Hz centres in a 2.4 kHz passband.

For each transmission, the application:

1. **Listens** — sample a short audio buffer, run the slot-energy
   survey to see which slots are occupied right now.
2. **Picks a random free slot** — uniform-random among the slots
   below an energy threshold.
3. **Transmits** at the picked audio centre.

This is **slotted ALOHA on the audio-frequency axis**, plus
listen-before-talk. It mirrors the natural amateur-radio "watch
the frequency, find a clear spot, transmit" practice — the
modem just formalises it inside the SSB audio passband. No
pre-assigned per-station slots; no central coordinator; no
elaborate state machines.

```
RF: 430.090 MHz USB  ←── one shared SSB channel ───→
    ┌──────────────────────────────────────────────┐
    │  audio passband (300 Hz … 2700 Hz)           │
    │                                              │
    │  ┌────────┐                  ┌────────┐      │
    │  │slot 1  │                  │slot 2  │      │
    │  │800 Hz  │                  │2000 Hz │      │
    │  └────────┘                  └────────┘      │
    │                                              │
    │   each TX: LBT → random pick from free slots │
    └──────────────────────────────────────────────┘
```

Collision behaviour with N slots, K simultaneous transmissions:
- Each transmitter independently picks a slot uniformly.
- Collision probability with one other ≈ 1/N. With N=2: 0.5.
- ARQ at the application layer handles retransmits; on a retry
  the transmitter re-runs LBT, so it converges quickly even at
  high contention.

## 2. Channel grid

A single uvpacket signal occupies ~1800 Hz end-to-end (RRC
roll-off included). Practical adjacent-slot separation for
< −20 dB inter-slot interference: **1200 Hz**.

| SSB passband | Slots | Slot centres (Hz) |
|---|---:|---|
| 2.4 kHz | 2 | 800, 2000 |
| 2.7 kHz | 2 | 850, 2150 |
| 3.0 kHz | 2 | 900, 2100 (or 700, 1900) |

For 0.3.3 the design assumes a 2-slot grid in 2.4 kHz SSB.
Wider SSB allocations are configurable via the band edges.

## 3. RX — `decode_multichannel`

### 3.1 API

```rust
/// Configuration for the multi-channel RX primitive and the
/// slot-occupancy survey helper.
#[derive(Clone, Copy, Debug)]
pub struct MultiChannelOpts {
    /// Low edge of the search band (Hz). Typical SSB:
    /// 300 Hz to clear the analog HPF.
    pub band_lo_hz: f32,
    /// High edge of the search band (Hz). Typical SSB:
    /// 2700 Hz for a 2.4 kHz passband.
    pub band_hi_hz: f32,
    /// Coarse-grid step in the frequency search (Hz).
    /// Default 25 Hz — same as the AFC coarse grid.
    pub coarse_step_hz: f32,
    /// Frequency-axis NMS radius (Hz) — peaks closer than this
    /// are merged. Default 600 Hz (= half slot spacing).
    pub nms_radius_hz: f32,
}

impl Default for MultiChannelOpts {
    fn default() -> Self {
        Self {
            band_lo_hz: 300.0,
            band_hi_hz: 2700.0,
            coarse_step_hz: 25.0,
            nms_radius_hz: 600.0,
        }
    }
}

/// Decode every uvpacket frame found in `audio` whose audio
/// centre lies in `[band_lo_hz, band_hi_hz]`. Returns
/// `(detected_audio_centre_hz, frame)` pairs — the centre is
/// reported so the receiver knows which slot each frame came
/// from (useful for matching ARQ ACKs to the original sender's
/// pending TX, even though the sender's slot is random per
/// transmission).
pub fn decode_multichannel(
    audio: &[f32],
    mc_opts: &MultiChannelOpts,
    fec_opts: &FecOpts,
) -> Vec<(f32, DecodedFrame)>;
```

### 3.2 Algorithm

```text
1. Coarse-grid scan
   for f in [band_lo_hz, band_hi_hz] step coarse_step_hz:
       mf  = down-convert + matched-filter(audio, f)
       (offset, mag) = best preamble peak in mf above threshold
       record (f, offset, mag) for each peak

2. Frequency-axis NMS
   sort peaks by magnitude descending
   greedily keep peaks ≥ nms_radius_hz apart in frequency
   (the same uvpacket signal lights up adjacent grid points
    by ~half a coarse-grid step; NMS collapses them)

3. Per-peak decode
   for each kept (f, offset):
       audio_offset = offset - SYM_PEAK_OFFSET
       for mode in [Robust, Standard, Fast, Express]:
           for n_blocks in (1..=32).rev():
               if decode_known_layout_with_opts(audio,
                       audio_offset, f, mode, n_blocks,
                       fec_opts).is_ok():
                   record (f, frame); break two

4. Time-axis dedup
   if two records have nearly identical (audio_offset, payload),
   keep the higher-magnitude one
```

The 25 Hz coarse-grid step leaves ≤ 12.5 Hz residual frequency
offset at the picked centre. The LMS quadratic phase fit inside
`decode_known_layout_with_opts` absorbs that. We use the
non-AFC entry point (no per-decode inner frequency search)
since the coarse-grid scan already gave us the centre.

### 3.3 Cost

Coarse scan: `(band_hi - band_lo) / 25 Hz` ≈ 96 candidates for
the default band. Each is one full down-convert + matched-filter
pass over the audio. ~70 ms in release for a 1-second audio
buffer.

Per-peak decode: typical 1–2 peaks × ~32 (mode × n_blocks)
attempts × ~10 ms each ≈ 0.3–1 s worst-case (BP + OSD-2).

Optimisation room (Phase 2 if needed): early-termination on
(mode, n_blocks); coarser frequency grid + finer refinement
around picked peaks.

## 4. TX-side helpers — slot survey + random picker

### 4.1 API

```rust
/// Per-slot received-signal-energy report.
#[derive(Clone, Copy, Debug)]
pub struct SlotEnergy {
    /// Slot centre frequency (Hz).
    pub audio_centre_hz: f32,
    /// Mean matched-filter output magnitude over the audio body.
    /// Higher = more uvpacket-like signal at this slot.
    pub mean_mf_magnitude: f32,
}

/// Measure the per-slot received-signal energy across the
/// configured band. Used by the LBT step before a random-slot TX.
///
/// Slots are placed at `band_lo_hz + slot_spacing_hz / 2`,
/// `band_lo_hz + 1.5 · slot_spacing_hz`, … up to `band_hi_hz`.
/// Energy is measured by matched-filtering `audio` at each slot
/// centre and averaging |mf|² over the audio body.
///
/// Policy-free: the helper just reports energies. The
/// application picks a slot via whatever rule fits.
pub fn measure_slot_energies(
    audio: &[f32],
    mc_opts: &MultiChannelOpts,
    slot_spacing_hz: f32,
) -> Vec<SlotEnergy>;
```

The application combines the survey with its own RNG to
implement the slotted-ALOHA step:

```rust
let slots = measure_slot_energies(&recent_audio, &mc, 1200.0);
// "Free" = at most 3 dB above band-median noise floor.
let median = slots
    .iter().map(|s| s.mean_mf_magnitude).collect::<Vec<_>>()
    .into_iter().fold(0.0_f32, |a, b| a.max(b))
    .max(1e-9);   // (replace with actual median computation)
let threshold = median * 2.0;
let free: Vec<f32> = slots
    .iter()
    .filter(|s| s.mean_mf_magnitude < threshold)
    .map(|s| s.audio_centre_hz)
    .collect();
let centre = match free.len() {
    0 => return Err(AllSlotsBusy),  // back off, retry later
    n => free[rng.gen_range(0..n)],
};
let burst = tx::encode(&header, &payload, centre)?;
transmit_via_radio(&burst).await;
```

mfsk-core ships **no** RNG dependency. The application supplies
its own randomness (`rand::Rng`, browser `crypto.getRandomValues`,
`/dev/urandom`, etc.). Keeps the modem layer side-effect-free.

### 4.2 Why MF-based survey

MF energy is sensitive specifically to narrowband signals near
the uvpacket modulation neighbourhood. A voice QSO from a
different SSB channel that's bleeding through nearby will light
up the matched filter weakly, while another uvpacket signal at
the slot centre lights it up strongly. This is the right bias
for the private-group slotted-ALOHA scenario.

For a public-channel deployment (mixed voice + data), an
FFT-based wideband energy detector would be preferred, but
that's a future cycle.

### 4.3 Cost

1 MF pass per slot (~1 ms each at 1 sec audio in release).
For a 2-slot grid: ~2 ms. Effectively free vs the RX scan.

## 5. Application-side workflow

```rust
// At each TX attempt:
let recent_audio = capture_audio_for(Duration::from_millis(500)).await;
let slots = measure_slot_energies(&recent_audio, &mc, 1200.0);
let centre = pick_random_free_slot(&slots, &mut rng)?;
let burst = tx::encode(&header, &payload, centre)?;
transmit_via_radio(&burst).await;

// Wait for ARQ ACK
match await_ack_with_timeout(&header.sequence, ack_window).await {
    Some(_) => { /* delivered */ }
    None => {
        // Collision (or weak signal). Retry with new random slot.
        sleep(rand_backoff()).await;
        retry();
    }
}

// At RX (continuous loop):
loop {
    let audio = capture_audio_for(Duration::from_millis(500)).await;
    for (sender_centre, frame) in decode_multichannel(&audio, &mc, &fec_opts) {
        // sender_centre identifies the slot the frame arrived in;
        // it's per-TX-random, so don't use it as a sender ID.
        // Sender identity comes from the application-layer payload
        // (e.g., an embedded callsign + signature).
        dispatch(frame);
        // Send ARQ ACK on a freshly-LBT-picked random slot.
    }
}
```

The application owns: capture/playback timing, ARQ ACK + retry
state machine, exponential backoff, voice-mode coexistence,
sender-identity authentication via signed payloads. mfsk-core
ships only the survey + decode primitives.

CSMA/**CD** proper isn't applicable to half-duplex SSB radio
(can't reliably listen while keying the transmitter). Slotted
ALOHA + LBT + ARQ at the application layer gives the same
collision-resilience behaviour with much less mechanism — which
is also why the design lines up so naturally with traditional
amateur-radio "watch then transmit" practice.

## 6. Test plan

### 6.1 RX

```rust
#[test]
fn decode_multichannel_finds_two_simultaneous_frames() {
    let burst_a = tx::encode(&header_a, &payload_a, 800.0).unwrap();
    let burst_b = tx::encode(&header_b, &payload_b, 2000.0).unwrap();
    let mut audio = vec![0.0_f32; max_len];
    add_at(&mut audio, &burst_a, 0);
    add_at(&mut audio, &burst_b, 100);
    let frames = decode_multichannel(&audio, &MultiChannelOpts::default(),
                                     &FecOpts::default());
    assert_eq!(frames.len(), 2);
    let frame_a = frames.iter().find(|(f, _)| (*f - 800.0).abs() < 50.0).unwrap();
    let frame_b = frames.iter().find(|(f, _)| (*f - 2000.0).abs() < 50.0).unwrap();
    assert_eq!(frame_a.1.payload[..payload_a.len()], payload_a[..]);
    assert_eq!(frame_b.1.payload[..payload_b.len()], payload_b[..]);
}
```

## 9. Out of scope

- ARQ / retransmit logic. Application-layer.
- RNG. Application-layer.
- Voice-mode coexistence (FFT-based wideband detector). Future
  cycle.
- Adaptive slot spacing. Stays at 1200 Hz.
- Wider-than-2.4-kHz SSB support beyond the configurable band
  edges (the algorithm handles it, the doc just doesn't dwell
  on it).
